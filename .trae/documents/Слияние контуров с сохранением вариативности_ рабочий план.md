## Цели
- Видимое слияние контуров в финальном `Akt[wght].ttf`
- Сохранение вариативности (`fvar/gvar`) и успешная сборка

## Стратегия
- Слияние до varLib (UFO-стадия, кубические кривые), без TT-правок
- Применять слияние только там, где топология после слияния совпадает во всех мастерах
- Не трогать последовательности точек точечным пером; запись обратно — только сегментным пером

## Этапы
1. Расширение компонент и построение пути
- Для каждого мастера создать `glyphSet`-отображение имя→глиф
- Строить контур через `PathOps.PathPen(base, glyphSet)` и `glyph.draw(PathPen)`

2. Слияние контуров
- Выполнить `pathops.simplify(base)` на кубике, получить объединённый путь

3. Проверка совместимости после слияния
- Для каждого мастера нарисовать объединённый путь во временный глиф и вычислить сигнатуру: `(кол-во контуров, длины точек по контурам)`
- Применять объединённые контуры назад в мастер только если сигнатуры совпадают у всех мастеров

4. Запись обратно в UFO
- Очистить контуры и компоненты, нарисовать объединённый путь сегментным пером (`getPen()`)
- Не выполнять реверсы направления/смену стартовой точки и не добавлять нулевые сегменты

5. Сборка VF
- Генерировать Designspace из `sources/Akt.glyphs`, заменить пути источников на модифицированные UFO
- Запустить `fontmake -m ... -o variable` (без TT-стадийного удаления наложений)

6. Валидация
- Проверить наличие `fvar/gvar`, диапазон оси `wght`
- Визуально подтвердить слияние на репрезентативных глифах

## Расширение охвата (после базового успеха)
- Канонизировать проблемные глифы в исходниках: выровнять число контуров и сегментацию во всех мастерах естественным образом (без «призрачных» контуров)
- После канонизации повторить шаги 1–5 для расширения слияния
